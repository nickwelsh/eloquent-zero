# Contributing

Thanks for contributing.

## Before opening a PR

- check existing issues and discussions first
- for bugs, include a minimal repro
- for new features, describe the use case before writing a large patch

## Local setup

```bash
composer install
vendor/bin/pest
```

## Pull requests

- keep PRs focused
- add or update tests when behavior changes
- update docs when public API or workflow changes
- keep code style consistent with the existing codebase

## Expectations

- breaking changes are possible right now
- maintainers may ask for changes before merge
- not every proposed feature will be accepted

## Questions

Use GitHub Discussions for questions and feature ideas when possible.
